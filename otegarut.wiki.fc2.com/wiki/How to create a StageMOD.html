How to create a StageMOD

日本語

Similar to CarMODS, you can make your own Stage by making and loading a 3D model.
The 3D file format that can be read is .obj (Wavefront Format).

Please look at the sample.

About Specifications(Reference)

It is necessary to set attributes for in-game physics calculations according to the loaded 3D model.
This is done automatically, but please note the object name of the 3D model for that.

Request

By including the following strings in the name of each 3D model, each property can be given.
Make sure that the names are all in uppercase.
Make sure those objects have meshes that are sized for the purpose.
Also, do not use the names specified below for unrelated objects.

Terrain System

Objects related to running on

ROAD

ground object
Please specify this where you want to run.

WALL

Wall objects
Please specify it as an object to stop the car from running through the object

DIRT

Dirt Object
Although it is the ground, please specify this for dirt to slow down the car

Judgement System

Collision objects are automatically made invisible (transparent).
Place it assuming that the judgment will be made when the car enters the range of the object.

SECTION

Please specify if you want to place a section (time measurement point) in the course.
The object will be invisible, so prepare an object separate from the appearance of the goal line.

MIZOHIT

This makes a gutter.
Having this box on top of a ditch will cause a ditch drop effect when a car touches it.

DOKO

"What the hell are you running on!?"
This makes the car go over the gutter and put the tires on the ground beside the gutter.

INBETA

"Upcoming Features"

Others

RIGIDBODY

The specified object is treated as a RigidBody.
For example, it can be used for pylons and soccer balls.

INVISIBLE

Objects with "INVISIBLE" in their names are made invisible(Transparent).

For example,
If the wall is named: "WALL_INVISIBLE" will be an invisible wall,
If the road is named:"ROAD_INVISIBLE" will create an invisible road.


Start position & direction(ver5.0.2)

STARTPOS

It will be the starting position of the car.
It is specified by a cube, and its center is the starting position.
The starting direction does not change unless "DIRECTION" is used.

DIRECTION

Used to specify the starting orientation of the car.
Cars are placed in the direction from STARTPOS to DIRECTION.
It is specified by a cube, Must be used with STARTPOS
