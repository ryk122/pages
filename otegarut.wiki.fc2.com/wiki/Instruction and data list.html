Instruction and data list

日本語

data form

【Premise】

In this game, the stage creates a path by arranging several map chips in order.
Other than LoadStage mode, random arrangement is performed.
In this mode, you can create your own stages by specifying the order in which the map chips appear.


【Overview】

It is a comma separated format. Read from the beginning of the text.
Data consists of instructions and map chips. Also supports comments (notes that do not affect the stage layout)

Map chip

Consists of three digits.
 

Detailed information


order

Starts with #. Place sections and perform special processing.

comment

Starts with%. However, since this is also data, it must be closed with "," (comma).
If you've ever created this kind of data, you'll know it, but it's useful if you can make notes.


grammar

Map chip
Three digits indicate map type, left / right, up / down.

Chip list

Detailed information

Map type

0: straight line
1: Curve 1
2: Curve 2 (over the groove)
3: Hairpin turn
4: Curve 3 (groove drop) (v3.6 corresponding later)
5: Bridge (Downhill only)
9: Rest area

Left and right

0: Right 1: Left

Top and bottom

0: Down 1: Up

Example)
For 100 mm, the map type indicates curve 1, right, and down.

order

Detailed information

#SECTION

Place the section pole. Section time can be measured.

#STOP

Place the barricade and prevent it from going further. Data can be written after this instruction.
Also, if you place it at the beginning of the data, you can prevent it from moving to the front of the starting point.

#CONE

Place a triangular cone. I wonder if it can be used for folding back.

#SCALExxx

If you write a number in xxx, the size of subsequent map chips will be xxx% times the size of the first one.
For example, # SCALE200 is twice the size = twice the road width, and # SCALE75 is 0.75 times the size.
It is also possible to put a space like "#SCALE 150 ,".
We Use it at Mt.Haruna.

#POFSXxxx #POFSYxxx #POFSZxxx

This is Position Offset. If you put a number (sign and decimal point are allowed) in "xxx,"
Next, shift the position of the map chip (including section and triangular cone) by "xxx" in the direction of the specified coordinate axis.
For example,
"000, # POFSY-1.5,000,"
In the case of, the joint between the straight (000) and the straight map tip moves -1.5 in the Y-axis direction, so there is a step with a height of 1.5.
Also, try using "Slalom", a stage example where a cone is placed in the middle of a map chip. (This is the main purpose order)
Note that movement is an absolute coordinate in the game. (Does not consider the direction of the road immediately before)

#ROFSXxxx #ROFSYxxx #ROFSZxxx

This is Rotation offset. A rotated version of the above POFS. We don't usually use it, but we hope that someone will do something use weird.

#OBJ (supported after ver.5.0.0)

Command for loading StageMOD.
As soon as this command is executed, the obj file loading screen will be displayed. Please check What is StageMOD
​​for details .

#OBJ <url> (supported since ver.5.0.0)

Used when combining stage data and StageMOD.
Also use this when using StageMOD ​​in event mode. (Not relevant to general players)
Load StageMOD ​​via web streaming.
For example, if you want to use the StageMOD
​​https://otegarut.wiki.fc2.com/sample.obj , use #OBJ https://otegarut.wiki.fc2.com/sample . See StageMOD ​​from URL for details .

#LAMPR #LAMPL　(ver.3.4 corresponding later)

You can put a street light on the way. If you add too much, the processing may be a little heavy.
Lights regardless of day or night. I will fix it soon.

#RAIN (supported after ver.4.29)

Change the weather to rain. The behavior of the car will also change.

#VS xx

Make CP cars appear in LoadStage mode.
Write it at the beginning of the stage data.
Also, the total course length is up to 1000 data.
If the xx part is -1, the car model is random, and if you write the number in VScp, you can specify the car.

#BLOOM xx (supported after ver.5.0.0)

This is a temporary workaround for lighting issues.
Solves the white glare problem when the weather is daytime setting.
Specifically, by inserting #BLOOM 1, the bloom expression is weakened and the glare is reduced.
It is assumed to be used in combination with StageMOD.

#CAMERAFAR xx (supported after ver.5.0.0)

It is a command to change the drawing distance of the camera.
If you prepare a large circuit with StageMOD ​​etc., it is used when the distant view is not drawn and it is unnatural.
Specifically, #CAMERAFAR 300. The standard value is about 100 to 400.
The higher the number, the heavier the processing of the game. Please be careful when using.

comment

Can be used by data designers to create notes.
It can be written as "% xxxxx". Be careful not to forget the trailing comma.


Confirmed bugs

In theory, it is possible to create orbiting stages,
Each time you place a part, the error accumulates and you may not always be able to place map chips in the same place after one lap.
